# PRD — Arquitetura SEO e Ordem de Implementação (SmartBus BR)


## Referência visual obrigatória

Esta página deve seguir o padrão visual definido em:

`docs/PRD/SEO/PRD — Padrão Visual das Páginas SEO.txt`

A página `/sistema-para-excursoes` é a página piloto oficial de referência visual para as páginas satélite SEO do SmartBus BR.

Em caso de dúvida sobre estrutura visual, hierarquia de blocos, cards, CTAs, uso de ícones, densidade visual ou organização da página, priorizar consistência com a página piloto.

## Objetivo

Definir a estrutura de criação, ligação e crescimento das páginas SEO do SmartBus BR, garantindo:

- consistência entre páginas
- ganho de autoridade no Google
- melhor experiência de navegação
- execução organizada no Lovable

---

## Conceito base

As páginas SEO devem funcionar em formato de cluster.

Cada cluster possui:

- 1 página principal (hub)
- páginas de apoio (topo, meio e fundo de funil)
- ligação entre todas as páginas

---

## Regra de interligação (CRÍTICO)

Todas as páginas devem seguir:

### 1. Linkar para a página hub do cluster

Exemplo:
- páginas de excursão → /sistema-para-excursoes

---

### 2. Linkar para páginas relacionadas do mesmo cluster

Exemplo:
- planilha → como organizar → controle → sistema

---

### 3. Linkar para outros clusters (leve)

Exemplo:
- excursão → caravanas → eventos

---

## Fluxo de navegação esperado

Topo → Meio → Fundo → Conversão

Exemplo:

como-organizar-excursao  
↓  
controle-de-passageiros  
↓  
sistema-gratuito  
↓  
sistema-para-excursoes

## Regras obrigatórias

- todas páginas devem ter navegação entre páginas
- manter padrão visual da landing
- não criar páginas isoladas
- sempre conectar com o cluster
- sempre ter CTA claro

---

## Critérios de sucesso

- páginas conectadas entre si
- fluxo lógico de navegação
- crescimento de SEO consistente
- aumento de tempo no site
- aumento de conversão

---

## Observações importantes

- este PRD define a ordem oficial de construção
- deve ser seguido antes de criar novas páginas
- novas páginas devem respeitar essa arquitetura
