# Documento Master – Como cadastrar um local de embarque

## 1. Objetivo do Processo
Cadastrar pontos de embarque para utilização nos eventos, estruturando origem dos passageiros, horários e organização da operação de saída.

## 2. Quando Utilizar
Utilize este fluxo quando:
- A empresa opera com novos pontos de embarque.
- Um local existente precisa de atualização de dados.
- Há necessidade de padronizar locais para uso recorrente em eventos.

## 3. Pré-requisitos
- Acesso administrativo com empresa ativa.
- Permissão para gestão de locais.
- Dados do ponto de embarque (nome, endereço, referência e contatos, quando aplicável).

## 4. Visão Geral do Processo
O cadastro é realizado na tela **Locais de Embarque**, com listagem e formulário de criação/edição. Após salvar, o local fica disponível para ser vinculado nos eventos e nas viagens.

## 5. Passo a Passo Completo
1. **Acessar Locais de Embarque**
   - **O que o usuário deve fazer:** entrar no menu “Locais”.
   - **O que o sistema faz automaticamente:** exibe os locais já cadastrados da empresa.
   - **O que deve ser conferido:** se o local desejado já existe para evitar duplicidade.

2. **Criar novo local**
   - **O que o usuário deve fazer:** clicar em “Adicionar Local” (ou ação equivalente de novo cadastro).
   - **O que o sistema faz automaticamente:** abre o formulário de local de embarque.
   - **O que deve ser conferido:** abertura correta do formulário.

3. **Preencher dados do local**
   - **O que o usuário deve fazer:** informar nome, endereço e informações auxiliares.
   - **O que o sistema faz automaticamente:** valida campos obrigatórios e registra o local.
   - **O que deve ser conferido:** clareza do endereço para orientação de passageiro e equipe.

4. **Salvar e validar disponibilidade**
   - **O que o usuário deve fazer:** salvar o cadastro.
   - **O que o sistema faz automaticamente:** inclui o local na lista da empresa.
   - **O que deve ser conferido:** se o novo local aparece para seleção em eventos.

5. **Editar ou inativar conforme necessidade**
   - **O que o usuário deve fazer:** atualizar dados quando houver mudanças operacionais.
   - **O que o sistema faz automaticamente:** persiste alterações e mantém histórico de uso.
   - **O que deve ser conferido:** se eventos futuros usarão o local correto.

## 6. Pontos de Atenção
- Nome do local deve ser objetivo e facilmente identificável.
- Endereço incompleto gera dúvidas no embarque.
- Evite criar variações com nomes muito parecidos para o mesmo ponto.

## 7. Boas Práticas
- Padronizar nomenclatura de locais por cidade/bairro.
- Incluir referências que facilitem localização no dia do evento.
- Revisar locais periodicamente antes de abrir vendas.

## 8. Impacto no Sistema
Locais de embarque bem cadastrados melhoram experiência do cliente, reduzem atrasos operacionais e organizam a logística das viagens. Também influenciam clareza na venda, no ticket e na validação de embarque.
