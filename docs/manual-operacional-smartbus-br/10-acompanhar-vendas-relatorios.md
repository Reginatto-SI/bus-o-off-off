# Documento Master – Como acompanhar vendas e relatórios

## 1. Objetivo do Processo
Monitorar desempenho comercial e operacional da empresa por meio das telas de vendas e relatórios, com visão de volume, status, receita e indicadores por evento.

## 2. Quando Utilizar
Utilize este fluxo quando:
- A gestão precisa acompanhar o desempenho diário/semanal/mensal.
- É necessário analisar vendas por evento, vendedor, status ou período.
- A equipe financeira precisa consolidar dados para conferência.

## 3. Pré-requisitos
- Acesso administrativo com empresa ativa.
- Eventos e vendas já registrados no sistema.
- Permissão para visualizar dados financeiros (quando aplicável ao perfil).

## 4. Visão Geral do Processo
O acompanhamento ocorre em duas frentes:
- **Vendas:** visão operacional com filtros, status, ações e detalhes de cada transação.
- **Relatório de Vendas:** visão analítica com abas de resumo e detalhado, além de exportação.

## 5. Passo a Passo Completo
1. **Acompanhar operação na tela de Vendas**
   - **O que o usuário deve fazer:** acessar “Vendas” para leitura rápida dos indicadores e lista.
   - **O que o sistema faz automaticamente:** mostra totais (vendas, pagas, reservadas, canceladas) e dados financeiros conforme perfil.
   - **O que deve ser conferido:** tendência de desempenho e possíveis desvios.

2. **Aplicar filtros operacionais**
   - **O que o usuário deve fazer:** filtrar por cliente, status, evento, vendedor e período.
   - **O que o sistema faz automaticamente:** atualiza a listagem conforme critérios informados.
   - **O que deve ser conferido:** se os resultados refletem corretamente o recorte desejado.

3. **Analisar detalhes de vendas específicas**
   - **O que o usuário deve fazer:** abrir detalhes de uma venda para investigar informações de cliente, assentos e histórico.
   - **O que o sistema faz automaticamente:** carrega tickets e logs da transação.
   - **O que deve ser conferido:** consistência de dados e status da venda.

4. **Usar Relatório de Vendas para visão gerencial**
   - **O que o usuário deve fazer:** acessar “Relatório de Vendas” e alternar entre abas de Resumo e Detalhado.
   - **O que o sistema faz automaticamente:** consolida dados por evento e detalha vendas individuais.
   - **O que deve ser conferido:** volume, cancelamentos, receita e custos por recorte.

5. **Refinar análises por período e responsáveis**
   - **O que o usuário deve fazer:** aplicar filtros por data, evento, vendedor e status também no relatório.
   - **O que o sistema faz automaticamente:** recalcula os dados com base no filtro aplicado.
   - **O que deve ser conferido:** comparabilidade entre períodos e metas internas.

6. **Exportar dados para uso externo**
   - **O que o usuário deve fazer:** acionar exportação em Excel/PDF quando necessário.
   - **O que o sistema faz automaticamente:** gera arquivo com os campos selecionados para auditoria/apresentação.
   - **O que deve ser conferido:** se o arquivo contém o mesmo recorte aplicado na tela.

## 6. Pontos de Atenção
- Análises sem filtro de período podem gerar leitura distorcida.
- Perfil sem permissão financeira pode não visualizar todos os indicadores.
- Cancelamentos devem ser acompanhados de perto para evitar erosão de receita.

## 7. Boas Práticas
- Definir rotina fixa de acompanhamento (diário e semanal).
- Comparar status de reservas versus pagas para ação comercial rápida.
- Exportar relatórios oficiais em datas de fechamento financeiro.
- Usar a aba resumo para decisão executiva e a detalhada para auditoria operacional.

## 8. Impacto no Sistema
O acompanhamento contínuo fortalece controle financeiro, previsibilidade de ocupação e qualidade de operação. Também apoia decisões de preço, divulgação, alocação de frota e gestão de equipe comercial.
